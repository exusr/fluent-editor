# Security Policy

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in FluentEditor, please report it responsibly.

### How to Report

- **Email**: Send a detailed report to the maintainers via the contact information in the [FUNDING.md](FUNDING.md) or repository profile.
- **GitHub Security Advisories**: If enabled for this repository, you may submit a private vulnerability report through GitHub's Security Advisory feature.

### What to Include

When reporting a vulnerability, please provide the following information:

- A clear description of the vulnerability
- Steps to reproduce the issue
- The affected version(s)
- Any potential impact or risk assessment
- Suggested fixes or mitigations (if any)

### Response Timeline

- **Acknowledgment**: We will acknowledge receipt of your report within 5 business days.
- **Investigation**: We will investigate and validate the vulnerability within 14 business days.
- **Resolution**: Once validated, we will work to release a fix and coordinate disclosure.
- **Disclosure**: We follow a coordinated disclosure policy and will not disclose the vulnerability until a fix is available or after a mutually agreed-upon timeframe.

## Security Best Practices for Users

- Always use the latest stable version of FluentEditor.
- Keep your Flutter SDK and dependencies up to date.
- When loading external documents, validate and sanitize input data.
- Be cautious when integrating with third-party services or plugins.

## Disclosure Policy

We ask that security researchers:

- Provide us with a reasonable amount of time to address the issue before disclosing it publicly.
- Make a good faith effort to avoid privacy violations and data destruction.
- Do not exploit the vulnerability beyond what is necessary to demonstrate the issue.

Thank you for helping keep FluentEditor and its users safe!
